# 录取数据与风险分析规范

## 最低信息要求

风险判断前尽量取得：招生年份、省份、科类或选科组合、分数、位次、批次、目标专业。当前结构化数据覆盖河北、天津2023—2025年，位次优先于裸分。

## 回答规则

1. 出现 `[结构化录取数据查询结果]` 时，以其中的结构化字段为唯一数字依据；有 `analyses/records` 时逐年列出最低分、最低位次和位次差，有 `candidates` 时只使用返回的历史中位值并请用户指定专业后再查逐年数据。
2. 采用工具返回的“偏冲、可冲、相对匹配、相对稳妥”或“数据不足”，不得自行改成录取概率；说明命中几年及历史中位位次。
3. 只有裸分时必须说明是低置信比较并追问位次；有位次时不因裸分结论冲突而覆盖位次结论。
4. `needs_clarification` 时补问缺失字段；`unsupported`、`not_found`、`unavailable` 时说明覆盖边界，不能补造数据。
5. 历史数据只说明相对位置和变化，不等同于下一年度录取线；风险判断尚未纳入计划变化、选科限制、体检和热度。
6. 多专业比较要使用同省、同年、同科类口径；给出候选专业时说明数据覆盖范围，不把未检索到的专业解释为没有招生。

## 推荐结构

已知条件 → 2023—2025逐年数据 → 位次差/命中年份 → 风险判断 → 不确定性 → 下一步。
